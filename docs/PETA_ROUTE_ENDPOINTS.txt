1. Route /register dan /register-guru di app.py dan /onboarding/register di Blueprint
   adalah DUA SISTEM REGISTRASI BERBEDA. Tim harus konfirmasi mana yang digunakan.

2. Ada route /auth/login DAN /login di app.py — DUPLIKASI. Pastikan tidak membingungkan
   user yang salah URL.

3. Semua route hapus (hapus-kelas, hapus-guru, hapus-siswa, delete halaqoh) menggunakan
   GET method — ini tidak ideal untuk operasi destructive. Standar REST menggunakan DELETE.
   [🟡 Perlu dicek apakah ada CSRF protection]

4. Route /admin/halaqoh/<id>/students menerima GET dan POST — cek apakah ada validasi
   CSRF token di form POST ini.
