# PETA DATABASE — EBSENSY ERP SAAS

## RULE PENTING PROJECT

### WAJIB

* Semua query HARUS menggunakan `school_id`
* Gunakan field asli database
* Gunakan relation yang sudah ada
* Gunakan repository/service pattern

### DILARANG

* Membuat table baru tanpa approval
* Rename field existing
* Hardcode data tenant
* Query tanpa tenant isolation
* Mengubah schema langsung tanpa migration

---

# DATABASE ARCHITECTURE

Project menggunakan:

* PostgreSQL
* Supabase
* Multi Tenant Architecture
* school_id isolation
* SaaS ERP Architecture

---

# TABLE: schools

## Fungsi

Master tenant sekolah.

## Relation

```txt
schools.id
→ semua table menggunakan school_id
```

## Rule

* Semua data HARUS terkait ke school_id
* Tidak boleh cross-tenant access

---

# TABLE: teachers

## Fungsi

Data guru / ustadz / pengajar.

## Relation

```txt
teachers.school_id → schools.id
teachers.subject_id → subjects.id
```

## Digunakan oleh

* absensi
* input nilai
* halaqoh
* dashboard guru
* raport

# RULE QUERY

## WAJIB

Contoh BENAR:

```sql
SELECT *
FROM students
WHERE school_id = %s
```

## SALAH

```sql
SELECT * FROM students
```

Karena:

* bisa bocor antar tenant
* melanggar multi-tenant isolation

---

# MIGRATION RULE

Semua perubahan database:

* WAJIB melalui migration SQL
* simpan di:

```txt
database/migrations/
```

Contoh:

```txt
01_create_teacher_accounts.sql
02_multi_role_system.sql
03_subject_based_system.sql
```
